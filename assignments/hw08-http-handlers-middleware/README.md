# Домашнее задание 8: HTTP-хэндлеры и middleware

## Цель

Научиться создавать HTTP-обработчики и middleware для расширения функциональности веб-серверов.

## Краткое описание задания

Вам нужно реализовать middleware для логирования и аутентификации, а также простой HTTP-обработчик.
Интерфейс `Logger` и сигнатуры функций заданы в `main.go`.

Необходимо:

1. Реализовать `LoggingMiddleware`:
   - оборачивает `http.Handler` и логирует каждый запрос
   - логируемая строка должна иметь вид `"<METHOD> <PATH>"`
   - логирование должно выполняться до вызова `next.ServeHTTP`

2. Реализовать `APIKeyAuthMiddleware`:
   - проверяет наличие и корректность ключа доступа в заголовке `X-API-Key`
   - если заголовок совпадает с `expectedKey`, запрос передаётся дальше
   - в противном случае возвращается статус `401 Unauthorized`, `next` не вызывается

3. Реализовать `PingHandler`:
   - отвечает на `GET /ping` статусом `200 OK` и телом `"pong"`
   - для остальных методов возвращает `405 Method Not Allowed`
