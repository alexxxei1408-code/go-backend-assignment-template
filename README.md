# Шаблон для Go-backend заданий

Это шаблонный репозиторий для курса Go-backend.

## Для студентов

1. Используйте этот шаблон для создания своего репозитория.
2. Реализуйте функции в файлах `main.go`.
3. Убедитесь, что тесты проходят: `go test -v`.
4. Помните, что публичные тесты (например, `main_test.go`) не покрывают все случаи. Итоговая проверка выполняется приватными тестами, поэтому ориентируйтесь на требования задания, а не на конкретные проверки тестов.

## Для ментора

Чтобы использовать в GitHub Classroom:

1. Создайте Assignment.
2. Выберите этот репозиторий как шаблон (Template repository).
3. Студенты получат копию с готовой структурой.

### Использование шаблона

**Вариант 1: Через GitHub Classroom (рекомендуется)**

1. Создайте Assignment в GitHub Classroom.
2. В качестве Template repository выберите `go-backend-assignment-template`.
3. При создании задания студенты получат полную копию этой структуры.

**Вариант 2: Ручное создание**

1. Студенты могут сами нажать "Use this template" на странице вашего репозитория.

### Важные настройки

**Защита ветки `main`**

В `Settings → Branches → Branch protection rules`:

- **Branch name pattern**: `main`
- **Require status checks to pass**: включите.
- **Require branches to be up to date before merging**: включите.

Это заставит студентов исправлять ошибки перед сдачей.

**Секреты для приватных тестов**

Если используете приватные тесты:

1. Перейдите в `Settings → Secrets and variables → Actions → New repository secret`.
2. Имя секрета: `PRIVATE_TESTS_TOKEN`.
3. Значение: ваш GitHub Personal Access Token.

Приватные тесты можно хранить в отдельном приватном репозитории и запускать их из workflow этого шаблона, используя `PRIVATE_TESTS_TOKEN`. Это позволяет скрыть реальные проверочные сценарии и не даёт студентам "подгонять" решения под известные тесты.

Теперь ваш шаблон готов к использованию в GitHub Classroom.
